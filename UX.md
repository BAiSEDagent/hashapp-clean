# UX

## Visual direction
- dark mode first
- Cash App-style financial UI
- soft cards, large numbers, clear spacing
- dense but readable activity list
- premium consumer feel, not enterprise tooling

## Main tabs
1. Home
2. Activity
3. Pay / Approvals
4. Payees
5. Agent / Rules

## Core screens
### Home
- balance
- active agent budgets
- pending approvals
- recent activity
- quick actions

### Activity
- chronological feed
- large amounts
- short event labels
- human-readable statuses
- tap into receipt detail

### Pay / Approval
- incoming purchase requests
- approve / deny / always allow under rule
- show reason, vendor, amount, policy match

### Payees
- search by ENS, Basename, address, agent name, service name
- trusted payees
- verified services
- avatars everywhere

### Agent detail
- linked identity
- active rules
- allowed vendors
- daily / per-tx limits
- pause / revoke

## UX thesis
The app should feel like a normal money app for abnormal agent behavior.
