What this actually is: A two-way messaging gateway. Any agent that has the API key can talk to the human. The human can talk back. About anything. Spend requests, market data, toast preferences — doesn't matter. The channel is open and general purpose.

The four pieces:

2. Agent sends messages inbound

POST /api/gateway/message
Authorization: Bearer <agent-api-key>
{
  "content": "anything the agent wants to say"
}
No enforced schema. The agent sends what it wants.

3. Frontend polls for new messages Simple interval — every 3 seconds check for new inbound agent messages. When one arrives it appears in the chat thread with the agent's name and avatar initial from onboarding.

4. Human replies go back to the agent

GET /api/gateway/messages?since=<timestamp>
Authorization: Bearer <agent-api-key>
The agent polls this to read human replies.

Venice stays but becomes optional If the agent wants private reasoning before sending a message, it uses Venice itself. Hashapp doesn't enforce it. The channel is transport — what the agent does before sending is its own business.
